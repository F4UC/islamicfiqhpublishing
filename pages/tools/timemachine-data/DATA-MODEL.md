# Time Machine — Data Model & Source Architecture

> เอกสารดีไซน์โครงสร้างข้อมูลของ Hijri Time Machine (R86 ingestion) — ออกแบบให้
> **รองรับการเพิ่มแหล่งอ้างอิงคลาสสิกจำนวนมากในอนาคตโดยไม่ต้องรีดีไซน์** ทุกอย่างเป็น
> DRAFT รอกองบรรณาธิการตรวจ · ตัวบทอาหรับ byte-exact (กฎ 60/86) · ทับศัพท์ผ่าน glossary (กฎ 86)

## 1. หลักการกลาง: ผูกแหล่งกับ "หน่วยข้อมูล" ที่ถูกต้อง (data granularity)

ตำราคลาสสิกมี **2 ธรรมชาติ** ที่ต่างกันโดยพื้นฐาน เก็บผิดที่จะทำให้ข้อมูลซ้ำซ้อน/บวม:

| ประเภท | ตัวอย่าง | หน่วยข้อมูล | ผูกกับ |
|---|---|---|---|
| **A. พงศาวดารรายปี** (annal / تاريخ على السنين) | อัลบิดายะฮ์ (อิบนุ กะษีร), อัลกามิล (อิบนุลอะษีร), อัลมุนตะซ็อม (อิบนุลเญาซีย์), **มิรอาตุซซะมาน (สิบฏ์ อิบนุลเญาซีย์)**, ตารีค อัฏฏ็อบรีย์ | "เหตุการณ์ในปี" | **EVENT** → `accounts[]` |
| **B. ตำราชีวประวัติ/ฏอบะกอต** (biographical dictionary / كتب التراجم والطبقات) | **ซิยัร อะอ์ลาม อันนุบะลาอ์ (อัซซะฮะบีย์)**, **ฏอบะกอต อัชชาฟิอียะฮ์ อัลกุบรอ (อัซซุบกีย์)**, ตะฮ์ซีบุลกะมาล (อัลมิซซีย์), วะฟะยาต อัลอะอ์ยาน (อิบนุ ค็อลลิกาน) | "ชีวิตบุคคล" | **PERSON** (glossary) → `tarajim[]` |

**เหตุผล:** พงศาวดารเล่า "เหตุการณ์เดียวกันข้ามปี" → เทียบเคียงได้ในแท็บของเหตุการณ์นั้น ·
ตำราชีวประวัติเล่า "คน" (ไม่มี 'เหตุการณ์ปี 132' มีแต่ 'ชีวประวัติอัสซัฟฟาห์') → ต้องผูกกับ
บุคคล แล้วตามบุคคลนั้นไปโผล่ทุกเหตุการณ์ที่เขาปรากฏ ใส่ครั้งเดียวใช้ได้ทั้งหมด

## 2. Source Registry กลาง — `sources.json`

ทุกแหล่ง (ทุกประเภท) ลงทะเบียน **ครั้งเดียว** ที่นี่ โดย key = `source id`:

```jsonc
"<source-id>": {
  "type": "annal" | "tarajim",   // กำหนดว่าผูกกับ event (annal) หรือ person (tarajim)
  "order": <int>,                // ลำดับแท็บ (ใช้เฉพาะ annal)
  "nameThai": "อิบนุ กะษีร",
  "book": "อัลบิดายะฮ์ วันนิฮายะฮ์",
  "bookArabic": "البداية والنهاية",
  "died": "ฮ.ศ. 774",
  "shamelaBook": 30097,          // (ออปชัน) ใช้ตรวจ/อ้างอิง; ลิงก์จริงเก็บราย account/tarajim
  "islamwebBook": 59             // (ออปชัน)
}
```

**กติกาเพิ่มแหล่งใหม่ในอนาคต = แก้ที่เดียว:** เพิ่ม 1 entry ใน `sources.json` →
renderer อ่านชื่อ/หนังสือจาก registry อัตโนมัติ ไม่ต้องแตะโค้ดหรือ schema เหตุการณ์

> ไฟล์เดิม `chronicles.json` ถูกแทนที่ด้วย `sources.json` (loader อ่าน `sources.json` ก่อน
> แล้ว fallback เป็น `chronicles.json`) — `sources.json` คือ source of truth ตั้งแต่นี้ไป

## 3. การผูกข้อมูล (binding)

### 3.1 Event-scoped (พงศาวดาร) — `event.accounts[]`
หนึ่ง entry ต่อหนึ่งแหล่งที่เล่าเหตุการณ์นี้ → renderer แสดงเป็น **แท็บ** (โชว์แท็บเมื่อ ≥ 2):
```jsonc
{ "source": "ibn-athir",          // ← อ้าง source id (เดิมใช้ key ชื่อ "chronicle" ยังรองรับ)
  "detail": "<คำแปลไทยเชิงเปรียบเทียบ>",
  "arabicExcerpt": "<ตัวบทอาหรับ byte-exact>",
  "loc": "เล่ม .. หน้า ..", "url": "<shamela>", "urlAlt": "<islamweb ถ้ามี>" }
```

### 3.2 Event-scoped (อ้างหน้าอย่างเดียว) — `event.crossRefs[]`
`{ "work": "...", "loc": "..." }` — ไม่มีคำแปล แสดงในบรรทัด "อ้างอิงเทียบเคียง"

### 3.3 Person-scoped (ตำราชีวประวัติ) — `glossary.persons[].tarajim[]`
```jsonc
"al-saffah": {
  "arabic": "...", "canonicalThai": "อัสซัฟฟาห์",
  "tarajim": [
    { "source": "al-dhahabi-siyar", "loc": "ج6/ص ...", "url": "https://shamela.ws/book/.../..." },
    { "source": "ibn-khallikan",     "loc": "ج.../ص ...", "url": "..." }
  ]
}
```
→ renderer แสดงเป็นลิงก์ชีวประวัติเล็กๆ ในบล็อก **"บุคคลสำคัญ"** ของทุกเหตุการณ์ที่บุคคลนี้ปรากฏ

### 3.4 เหตุการณ์ "ผู้วายชนม์" (وفيات) — สะพานเชื่อม A↔B
เหตุการณ์ประเภทนี้คือชีวประวัติคนอยู่แล้ว จึงอนุญาตให้ tarajim source ขึ้นเป็น **account/แท็บ**
ได้ด้วย (เช่น แท็บ "ซิยัร" ที่มี excerpt ชีวประวัติ byte-exact) — เข้ากันทั้งสองแบบ

## 4. Logic ที่รองรับข้อมูลมหาศาล (scalability)

1. **Registry-driven:** renderer ไม่ hardcode ชื่อแหล่งใดๆ — อ่านจาก `sources.json` ทั้งหมด
2. **ไม่ระเบิดข้อมูล:** ข้อมูลระดับคน (tarajim) ใส่ครั้งเดียวต่อบุคคล ใช้ซ้ำทุกเหตุการณ์ที่คนนั้นโผล่
   (ต่างจากการใส่ซ้ำทุกเหตุการณ์) · ข้อมูลระดับเหตุการณ์ (accounts) เพิ่มทีละแหล่งตามต้องการ
3. **เพิ่มได้ไม่จำกัด:** N พงศาวดาร → N แท็บ · M ตำราชีวประวัติ → M ลิงก์ต่อบุคคล
4. **Backward-compatible:** เหตุการณ์/บุคคลที่ยังไม่มีแหล่งใหม่ ก็เรนเดอร์ปกติ (graceful degrade)
5. **Byte-exact + glossary lock:** ทุกตัวบทอาหรับคัดลอกตรงจากแหล่ง (กฎ 60/7/8) ทับศัพท์ตาม `canonicalThai` (กฎ 86)
