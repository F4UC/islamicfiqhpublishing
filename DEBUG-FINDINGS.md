### 1. Site-wide nav links 308-redirect (perf + canonical consistency) â€” highest value
The global `components/header.html`, `components/footer.html`, and the homepage
nav hard-code `.html` hrefs (`/pages/kalam.html`, `/index.html`, â€¦). Cloudflare
serves clean URLs, so each returns **`308 â†’ /pages/kalam`**. Every nav/footer
click site-wide pays an extra redirect hop, and it contradicts `main.js`, which
deliberately strips `.html` to "canonical extensionless URLs."
- **Where:** `components/header.html` (8 hrefs), `components/footer.html` (8),
  homepage nav (8).
- **Fix:** drop the `.html` suffix from those static hrefs.
- **Verify:** on the PR preview, nav links return `200` directly (no `308`).

### 2. Latent bug in `functions/api/collect.js` (view counter)
`VALID_IDS` is built only from the **array** form of `articles.json`:
```js
const VALID_IDS = new Set((Array.isArray(articlesData) ? articlesData : []).map(a => a && a.id).filter(Boolean));
```
CI's parser handles both shapes (`Array.isArray(a) ? a : (a.articles || Object.values(a)[0])`).
Today `articles.json` is an array, so this works. But if it is ever wrapped
(e.g. `{ "articles": [...] }`), `VALID_IDS` becomes **empty**, every `article_id`
fails validation, and **all view counts silently drop to zero** (still `204`, so
no error surfaces).
- **Fix:** mirror CI's defensive parse so both shapes are accepted.
