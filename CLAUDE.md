# Islamic Fiqh Publishing — Editorial Charter (Universal Agent Rules)

> **MANDATORY:** Every AI agent working on this repository must read and comply with all 51 rules below before executing any task. These rules are the single source of truth for all editorial, typographic, and technical standards.

# ธรรมนูญกองบรรณาธิการ Islamic Fiqh Publishing (51 กฎเหล็ก)

### หมวดที่ 1: มาตรฐานวิชาการและความสมบูรณ์ของเนื้อหา (Editorial & Academic Integrity)

* **กฎข้อที่ 1:** คงเนื้อหาต้นฉบับไว้ครบถ้วน 100% ห้ามตัดทอน ย่อความ หรือดัดแปลงสารัตถะหลักโดยไม่ได้รับอนุมัติจากกองบรรณาธิการ
* **กฎข้อที่ 2:** การปรับแก้ซอร์สโค้ดในตัวบทความ ให้กระทำได้เฉพาะส่วนโครงสร้าง HTML และการจัดแต่ง CSS เพื่อรองรับระบบแสดงผลที่ถูกต้องเท่านั้น
* **กฎข้อที่ 3:** ห้ามต่อเติมข้อมูลที่มาจากการคาดเดาหรือขาดหลักฐานอ้างอิงในเชิงวิชาการอิสลาม (นิติศาสตร์, เทววิทยา, หะดีษ, ประวัติศาสตร์)
* **กฎข้อที่ 4:** เนื้อหาบทความและการวิเคราะห์ต้องอ้างอิงตำราแม่บท (Primary Sources) ยุคคลาสสิกอย่างชัดเจนและถูกต้องตามระเบียบวิธีวิจัย
* **กฎข้อที่ 5:** หลีกเลี่ยงการใช้คำราชาศัพท์หรือภาษาลำลองในการกล่าวถึงท่านศาสดาและบรรดาปราชญ์ ให้ใช้ภาษาเขียนเชิงวิชาการที่เหมาะสม
* **กฎข้อที่ 6:** รายการหนังสือภาษาอาหรับในส่วนบรรณานุกรม ต้องใช้ภาษาอาหรับต้นฉบับเท่านั้น ห้ามแปลชื่อหนังสือเป็นภาษาอื่นเพื่อรักษาอัตลักษณ์ของตำรา
* **กฎข้อที่ 7:** **[การสอบทานอัลกุรอาน]** หากมีการอ้างอิงโองการอัลกุรอาน ต้องดึงตัวบทมาจาก `https://quran.com/ar` เพื่อความแม่นยำของอักขระและสระ พร้อมระบุชื่อสูเราะฮ์และหมายเลขอายะฮ์กำกับให้ถูกต้องทุกครั้ง
* **กฎข้อที่ 8:** **[การสอบทานหะดีษ]** หากมีการอ้างอิงตัวบทหะดีษ ต้องดึงข้อมูลจาก `https://sunnah.com/` เป็นหลัก หากไม่มีให้ค้นคลังข้อมูล `https://shamela.ws/` และ **ต้องใส่สระกำกับตัวบทให้ครบถ้วนทุกครั้ง** หากค้นไม่พบจากทั้งสองแหล่ง จึงจะอนุญาตให้ยึดตามต้นฉบับร่างของผู้เขียนได้

### หมวดที่ 2: มาตรฐานอักขรวิธี การทับศัพท์ และการเลือกใช้คำ (Transliteration & Word Choice)

* **กฎข้อที่ 9:** การระบุชื่อปราชญ์ในครั้งแรกของบทความ จำเป็นต้องระบุปีเสียชีวิตในวงเล็บตามรูปแบบมาตรฐาน: `(เสียชีวิตปี ฮ.ศ. XXX / ค.ศ. XXX)`
* **กฎข้อที่ 10:** การใส่คำสดุดีท้ายนามปราชญ์ (เช่น เราะหิมะฮุลลอฮ์, เราะฎิยัลลอฮุอันฮุ) ให้วางต่อท้ายชื่อได้ทันที **ห้ามใส่วงเล็บครอบคำสดุดีโดยเด็ดขาด** แม้จะวางคู่กับปีเสียชีวิตก็ตาม
* **กฎข้อที่ 11:** การทับศัพท์คำนามภาษาอาหรับที่มี อัล- (ال) นำหน้า ให้สะกดตามการเชื่อมสระและเสียงพยัญชนะชัมซียะฮ์อย่างถูกต้อง ยกเว้นคำว่า **"ชะรีอะฮ์"** ให้ตัดคำว่า "อัช" ออกไป เน้นเฉพาะคำนี้เท่านั้น
* **กฎข้อที่ 12:** การถอดเสียงชื่อเฉพาะทางวิชาการและชื่อปราชญ์ ต้องอิงตามอักขรวิธีภาษาอาหรับมาตรฐาน (เช่น อัลเฆาะซาลีย์, อัลมาตูรีดีย์, อัซซุบกีย์) และหลีกเลี่ยงการสะกดว่า "ท่านอิมาม" หรือ "อิมาม" ให้ใช้คำว่า "อิหม่าม" แทนในทุกกรณี
* **กฎข้อที่ 12.1:** **[การทับศัพท์คำว่า صحيح]** การถอดเสียงคำว่า **صحيح** (Sahih) เป็นภาษาไทย ต้องสะกดว่า **"ศอเฮี้ยะฮ์"** เท่านั้น (เช่น ศอเฮี้ยะฮ์อัลบุคอรีย์, ศอเฮี้ยะฮ์มุสลิม) **ห้ามสะกดว่า "ศ่อเหี้ยห์" โดยเด็ดขาด** และให้ใช้รูปแบบนี้อย่างสม่ำเสมอในเนื้อหาทั้งหมดในอนาคต
* **กฎข้อที่ 13:** **[ช่องไฟคำทับศัพท์]** คำศัพท์เทคนิคทางศาสนาที่ทับศัพท์มา เมื่อนำมาประกอบในประโยคภาษาไทย **ห้ามเว้นวรรคหน้า-หลังเด็ดขาด** (เช่น ให้เขียนว่า `ฐานหลักของชะรีอะฮ์ในประเด็นนี้` ห้ามเขียนแยกเป็น `ฐานหลักของ ชะรีอะฮ์ ในประเด็นนี้`)
* **กฎข้อที่ 14:** **[การเลือกใช้คำเชื่อม]** ห้ามใช้คำเชื่อมโบราณเช่นคำว่า **"ทว่า"** ให้เปลี่ยนไปใช้คำว่า **"แต่"** แทนในทุกกรณี เพื่อความกระชับของภาษาวิชาการ
* **กฎข้อที่ 15:** การอ้างอิงคำพูดของนักวิชาการ, ตัวบทหะดีษ หรือคำแปล ต้องใช้เครื่องหมายอัญประกาศแบบโค้ง **`"..."`** เท่านั้น ห้ามใช้แบบตรง (`"..."`) เพื่อความพรีเมียม
* **กฎข้อที่ 16:** ชื่อตำราและหนังสือที่ปรากฏในเนื้อหาบทความและบรรณานุกรม ต้องกำกับด้วยแท็กตัวหนา `<strong>` เสมอ เพื่อเน้นย้ำความสำคัญ
* **กฎข้อที่ 17:** ห้ามใช้แท็กตัวเอียง `<em>` หรือตัวเอียงในระบบ CSS กับชื่อตำราและหนังสือ ให้สงวนไว้สำหรับคำเน้นย้ำทั่วไปที่ไม่ใช่ชื่อเฉพาะ

### หมวดที่ 3: โครงสร้างภาษาและสถาปัตยกรรมระดับบล็อก (Language & Typography Structure)

* **กฎข้อที่ 18:** โครงสร้างหลักแท็ก `<body>` และแท็กเนื้อหาภาษาไทยทุกจุด ต้องระบุแอตทริบิวต์ `lang="th"` เพื่อมาตรฐาน Accessibility และระบบบอทดัชนี
* **กฎข้อที่ 19:** **[สมดุลตัวบทอาหรับ]** หากข้อความภาษาอาหรับมีความยาว **ไม่เกิน 1 บรรทัด** ให้จัดกึ่งกลาง (Center Layout) แต่หากยาว **เกิน 1 บรรทัดขึ้นไป** ให้จัดเข้าสู่กฎบล็อกอ้างอิง (Blockquote) ตามปกติ
* **กฎข้อที่ 20:** การจัดวางคำแปลภาษาไทยของตัวบทอ้างอิงภาษาอาหรับ **ในกรณีที่ตัวบทมีความยาวไม่เกิน 60 คำ** ต้องวางแยกบรรทัดไว้ใต้ตัวบทอาหรับ และห่อหุ้มด้วยแท็ก `<div class="thai-quote">` เสมอ (สำหรับตัวบทที่ยาวเกิน 60 คำ ให้ปฏิบัติตามกฎข้อที่ 49)
* **กฎข้อที่ 21:** การเน้นคำสำคัญ (Highlight) ที่มีน้ำหนักวิชาการภายในย่อหน้าบทความ ต้องใช้แท็ก `<strong>` เท่านั้น ห้ามใช้โครงสร้างอื่นใด
* **กฎข้อที่ 22:** รูปแบบโครงสร้างความสัมพันธ์ระหว่างตัวบทอาหรับและคำแปลไทยในกล่องฟิกฮ์ ต้องควบคุมผ่านคลาสสถาปัตยกรรมระบบที่เสถียร

### หมวดที่ 4: สถาปัตยกรรมซอร์สโค้ดและระบบสไตล์ชีต (Code & CSS Grid Standards)

* **กฎข้อที่ 23:** สไตล์ CSS ทั้งหมดต้องถูกประกาศและรวบรวมไว้ภายในแท็ก `<style>` ภายใต้โครงสร้าง `<head>` เท่านั้น ห้ามใส่แทรกกลาง `<body>` โดยเด็ดขาด
* **กฎข้อที่ 24:** ฟอนต์ตัวพิมพ์ต้องยึดมาตรฐานดังนี้: ภาษาอาหรับใช้ **Amiri**, ภาษาไทยใช้ **Sarabun** และภาษาอังกฤษใช้ **Inter** ห้ามใช้ฟอนต์อื่นเด็ดขาด
* **กฎข้อที่ 25:** ทุกหน้าเว็บต้องระบุแอนเคอร์แท็ก `id="header-placeholder"` และ `id="footer-placeholder"` ในตำแหน่งที่ถูกต้อง เพื่อให้ระบบสคริปต์กลางเรียกโมดูลมาแสดงผลได้สมบูรณ์
* **กฎข้อที่ 26:** การเชื่อมโยงเส้นทางไฟล์ (Path Reference) ของ CSS, JS, และ Assets สำหรับไฟล์ที่อยู่ในโฟลเดอร์ย่อย ต้องใช้ระบบถอยหลังสองชั้น (`../../`) อย่างเป็นระบบ
* **กฎข้อที่ 27:** กำหนดระยะห่างระหว่างบรรทัดในเนื้อหาบทความด้วย `line-height: 2.0` เป็นมาตรฐาน เพื่อรองรับรูปสระภาษาไทยและเครื่องหมายสระภาษาอาหรับไม่ให้ซ้อนทับกัน
* **กฎข้อที่ 28:** **[มาตรฐานช่องไฟภาษาไทย]** ห้ามใช้คำสั่ง `text-align: justify;` ในย่อหน้าบทความโดยเด็ดขาด ให้บังคับใช้ `text-align: left;` คู่กับ `word-break: break-word;` เพื่อป้องกันปัญหาช่องว่างระหว่างคำผิดธรรมชาติ
* **กฎข้อที่ 29:** ใช้รหัสสีทองพรีเมียม `#c49a45` สำหรับป้ายสถานะ ปุ่ม และองค์ประกอบไฮไลต์ เพื่อควบคุมอัตลักษณ์ของแบรนด์
* **กฎข้อที่ 30:** ห้ามใช้ `Inline Style` (การเขียนสไตล์ในแท็ก HTML โดยตรง) ในทุกกรณี เว้นแต่เป็นสไตล์ที่ควบคุมผ่านคำสั่งสคริปต์ที่มีการคำนวณค่าตัวแปร
* **กฎข้อที่ 31:** โครงสร้างข้อมูลและคำสั่งในการสุ่มวาทะปราชญ์ (Quote of the Day) ต้องถูกจัดวางอย่างเป็นระเบียบในส่วนท้ายของไฟล์ก่อนปิดแท็ก `</body>`

### หมวดที่ 5: ประสบการณ์ผู้ใช้และมิติดิจิทัลอินเตอร์แอคทีฟ (UI/UX & Digital Experience)

* **กฎข้อที่ 32:** **[สถาปัตยกรรมหน้าอ่านบทความ]** ยกเลิกระบบสารบัญอัจฉริยะ (Sidebar TOC) ทั้งหมด และบังคับใช้ **Centered Reading Layout** โดยจำกัดความกว้างสูงสุดไว้ที่ `900px` เพื่อให้ผู้อ่านโฟกัสเนื้อหาได้ 100%
* **กฎข้อที่ 33:** หน้าแผงควบคุมเครื่องมือ (`tools.html`) ต้องใช้ดีไซน์ Premium Minimalist เน้นการ์ดสีขาวสะอาดตา ขอบบาง และใช้เงานุ่มนวล
* **กฎข้อที่ 34:** การ์ดแสดงผลเครื่องมือและแอปพลิเคชันต้องระบุป้ายสถานะ (Status Badge) เสมอ เช่น "พร้อมใช้งาน", "กำลังพัฒนา" หรือ "กำลังพิจารณา"
* **กฎข้อที่ 35:** ระบบเว็บต้องเปิดใช้งานแอนิเมชันลอยขึ้นเมื่อเลื่อนจอ (`Scroll Reveal`) โดยใช้คลาส `.reveal`
* **กฎข้อที่ 36:** ต้องติดตั้งปุ่มกลับขึ้นบนสุด (`#backToTopBtn`) ในตำแหน่ง Fixed Position ที่จะปรากฏขึ้นอัตโนมัติเมื่อเลื่อนจอลงเกิน 400px
* **กฎข้อที่ 37:** ระบบโครงสร้าง CSS ต้องรองรับการสลับเป็นโหมดมืด (Dark Mode) ผ่านคลาส `.dark-mode` ที่บอดี้
* **กฎข้อที่ 38:** แผงเครื่องมือช่วยอ่าน ต้องมีฟังก์ชันปรับขนาดตัวอักษรที่ทำงานได้จริง (A-, Normal, A+) โดยจำกัดขนาดระหว่าง 14px ถึง 28px
* **กฎข้อที่ 39:** ติดตั้งแถบแสดงความคืบหน้าในการอ่าน (Reading Progress Bar) บนสุดของหน้าบทความ โดยคำนวณเปอร์เซ็นต์ตามการเลื่อนหน้าจอจริง
* **กฎข้อที่ 40:** **[มาตรฐานบรรณานุกรม]** การคัดลอกอ้างอิง ต้องใช้ฟอร์แมต: `ชื่อผู้เขียน/กองบรรณาธิการ. (ปี พ.ศ.). ชื่อบทความ. Islamic Fiqh Publishing. สืบค้นจาก [URL]`
* **กฎข้อที่ 41:** ปุ่มคัดลอกข้อความอ้างอิงและลิงก์ ต้องมีแจ้งเตือน "คัดลอกแล้ว ✓" เป็นเวลา 2 วินาที (Micro-interaction)
* **กฎข้อที่ 42:** รูปภาพและ Assets ทุกชิ้นต้องมีการระบุแอตทริบิวต์ `alt` ที่อธิบายความหมายสอดคล้องกับภาพ
* **กฎข้อที่ 43:** การจัดกล่องเนื้อหาอ้างอิงในบทความนิติศาสตร์ ต้องแยกแยะระหว่างตัวบทอาหรับบริสุทธิ์และส่วนแปลไทยเด็ดขาดชัดเจน โดยยึดตามมาตรฐานของกฎข้อที่ 20 (กรณีตัวบทไม่เกิน 60 คำ) และกฎข้อที่ 49 (กรณีตัวบทเกิน 60 คำ)
* **กฎข้อที่ 44:** การตั้งชื่อหน้าเว็บในแท็ก `<title>` ต้องเป็นไปในแนวทางเดียวกัน คือ `[ชื่อบทความ/ชื่อหน้า] — Islamic Fiqh Publishing` โดยใช้ขีดข้างยาว (Em Dash)
* **กฎข้อที่ 45:** โครงสร้าง Layout ต้องได้รับการทดสอบและเขียน Media Queries (`@media`) ให้แสดงผลแบบ Responsive บนมือถืออย่างสมบูรณ์
* **กฎข้อที่ 46:** หน้าแรก (`index.html`) ต้องมีส่วนดึง "แอปพลิเคชันแนะนำ" แสดงผลในตำแหน่งก่อนเข้าสู่หมวดหมู่บทความล่าสุด

* **กฎข้อที่ 47:** **มาตรฐานการกำหนดทิศทางข้อความอาหรับ (Right-to-Left / RTL)**
เพื่อป้องกันปัญหาการแสดงผลเครื่องหมายวรรคตอนผิดเพี้ยน (เช่น จุดมหัพภาคกระโดด) และให้ Bullet Points อยู่ในตำแหน่งที่ถูกต้อง ให้บังคับใช้แอตทริบิวต์ `dir="rtl"` เสมอ ควบคู่กับ `lang="ar"` ในแท็กระดับบล็อก (Block-level) หรือรายการ (Lists) ที่บรรจุข้อความภาษาอาหรับล้วน
* **กฎข้อที่ 48:** **[ตัวอย่างโค้ดมาตรฐานระดับบล็อกอ้างอิงและบรรณานุกรม]**
```html
<ul class="ref-list" lang="ar" dir="rtl">
   <li>...เนื้อหาภาษาอาหรับ...</li>
</ul>

<blockquote lang="ar" dir="rtl" class="quote-container">
   <span class="ar">...เนื้อหาภาษาอาหรับ...</span>
</blockquote>

<div class="references">
 <h2 class="ref-title" lang="th">บรรณานุกรม</h2>
 <ul class="ref-list" lang="ar" dir="rtl">
   <li>ابن كثير، عماد الدين إسماعيل. <strong>البداية والنهاية</strong>. بيروت: دار إحياء التراث العربي.</li>
   <li>البخاري، محمد بن إسماعيل. <strong>صحيح البخاري</strong>. بيروت: دار طوق النجاة.</li>
 </ul>
</div>
```

### หมวดที่ 7: มาตรฐานการแสดงผลตัวบทยาวและอักษรผสม (Long-form & Mixed Script Standards)

* **กฎข้อที่ 49:** **[มาตรฐานตัวบทยาว — การนำเสนอและการระบุเจ้าของคำพูด]** หากตัวบทต้นฉบับ (ภาษาอาหรับหรือภาษาอื่น) มีความยาวเกิน **60 คำ** ให้ปฏิบัติดังนี้:
  1. แสดงเฉพาะ**คำแปลภาษาไทย**เป็นหลัก โดยต้องขึ้นต้นด้วยสูตร **attribution** ตามประเภทของแหล่งอ้างอิง เพื่อให้ผู้อ่านทราบชัดเจนว่านี่คือคำพูดของใคร ไม่ใช่ของผู้เขียนบทความ:
     - **คำพูดปราชญ์:** `[ชื่อ] (เสียชีวิตปี ฮ.ศ. XXX / ค.ศ. XXX) [คำสดุดี] กล่าวว่า:` เช่น `อัซซุบกีย์ (เสียชีวิตปี ฮ.ศ. 771 / ค.ศ. 1370) เราะหิมะฮุลลอฮ์ กล่าวว่า:`
     - **โองการอัลกุรอาน:** `อัลลอฮ์ ซุบหานะฮูวะตะอาลา ตรัสในสูเราะฮ์ [ชื่อสูเราะฮ์] อายะฮ์ที่ [X] ว่า:`
     - **หะดีษ:** `ท่านนบีมุหัมมัด ศ็อลลัลลอฮุอะลัยฮิวะซัลลัม กล่าวว่า:` (บันทึกโดย [ชื่อผู้บันทึก เช่น อัลบุคอรีย์, มุสลิม])
  2. ซ่อนตัวบทต้นฉบับไว้ภายใต้แท็ก `<details>/<summary>` โดยใช้ป้ายกำกับ **"ดูตัวบทต้นฉบับ ▼"** วางไว้ใต้คำแปลไทยภายใน blockquote เดียวกัน

```html
<blockquote class="quote-container">
  <div class="thai-quote">
    <span class="attribution">อัซซุบกีย์ (เสียชีวิตปี ฮ.ศ. 771 / ค.ศ. 1370) เราะหิมะฮุลลอฮ์ กล่าวว่า:</span>
    "...คำแปลภาษาไทย..."
  </div>
  <details class="source-toggle">
    <summary>ดูตัวบทต้นฉบับ ▼</summary>
    <div lang="ar" dir="rtl" class="ar">...نص عربي...</div>
  </details>
</blockquote>
```

* **กฎข้อที่ 50:** **[มาตรฐานสมดุลอักษรอาหรับในบรรทัดผสม — Inline Arabic Balance]** เมื่ออักษรอาหรับปรากฏแทรกอยู่ในบรรทัดภาษาไทย ต้องปรับสมดุลการแสดงผลผ่าน CSS ดังนี้:
  - **สัดส่วนขนาดอักษร:** กำหนด `font-size: 1.15em` สำหรับ `:lang(ar)` เพื่อให้ตัวอักษร Amiri มีขนาดสายตาเทียบเท่ากับ Sarabun
  - **ตำแหน่งแนวตั้ง:** กำหนด `vertical-align: middle` สำหรับ `span.ar-inline` เพื่อให้เสมอกัน
  - **ความสูงบรรทัด:** กำหนด `line-height: 2.4` สำหรับย่อหน้าที่มีส่วนผสมอาหรับ เพื่อป้องกันสระอาหรับซ้อนทับ

* **กฎข้อที่ 51:** **[มาตรฐานส่วน QUOTE OF THE DAY]** ระบบวาทะปราชญ์ประจำวัน (QOTD) ต้องปฏิบัติตามข้อกำหนดต่อไปนี้อย่างเคร่งครัด:
  - **ความถี่ในการสลับ:** `setInterval` ใช้ค่า **9,000ms (9 วินาที)** เป็นมาตรฐาน ห้ามเปลี่ยนแปลงค่านี้โดยไม่ได้รับอนุมัติจากกองบรรณาธิการ
  - **มาตรฐานตัวบทอาหรับ:** ข้อความอาหรับทุกรายการต้องใส่สระกำกับ (Harakat) ครบถ้วน และต้องระบุ `lang="ar" dir="rtl"` บน element ที่บรรจุตัวบท ห้ามใส่อัญประกาศหรือจุด (.) ในฟิลด์ `ar` โดยเด็ดขาด
  - **[สำคัญมาก] ความถูกต้องและสมบูรณ์ของตัวบท:** หากผู้เขียน/กองบรรณาธิการ **ระบุตัวบทมาให้โดยตรง** ให้ยึดตามนั้นทุกประการโดยไม่ต้องค้นหาหรือเปลี่ยนแปลงสำนวน แต่ยังต้องปฏิบัติ 2 ข้อนี้: (1) **ใส่สระกำกับ (Harakat)** ให้ครบถ้วนหากยังไม่มี และ (2) **ระบุแหล่งอ้างอิง** (ชื่อหนังสือ/แหล่งข้อมูล) กำกับไว้ทุกครั้ง — แต่หากไม่มีการระบุมาให้ ต้องค้นหาจากอินเทอร์เน็ตและแหล่งปฐมภูมิเพื่อยืนยันว่าตัวบทไม่ถูกตัดตอน ไม่มีคำเกิน และการระบุผู้กล่าวถูกต้อง
  - **มาตรฐานคำแปลภาษาไทย:** คำแปลต้องใช้ภาษาธรรมชาติที่สอดคล้องกับโทนของแหล่งที่มา (กวี, วรรณกรรม, เทววิทยา, หรือการเมือง) — ถูกต้อง, เข้าใจง่าย, กระชับ; ใช้ `\n` ในตำแหน่งที่สอดคล้องกับจุดหยุดธรรมชาติเพื่อควบคุมการตัดบรรทัดบนมือถือ
  - **การแสดงผลบนมือถือ:** ใช้ `<br>` แทน `&nbsp;` ในการแบ่งวรรคกวีอาหรับ เพื่อให้แต่ละวรรคแสดงในบรรทัดใหม่บนหน้าจอเล็ก (ยกเว้นกรณีที่ทดสอบแล้วว่าแสดงผลได้สมบูรณ์โดยไม่ต้องแบ่ง)
  - **โครงสร้างข้อมูล:** array `quotes` ต้องประกอบด้วย field `ar` (ตัวบทอาหรับ ไม่มีอัญประกาศเป็นเนื้อหา), `th` (คำแปลไทยที่ล้อมด้วย `"..."`), และ `author` (ชื่อผู้กล่าวและปีเสียชีวิตตามรูปแบบ `ชื่อ · เสียชีวิตปี ฮ.ศ. XXX` ไม่มีอัญประกาศ) และต้องวางไว้ก่อนปิดแท็ก `</body>` ตามกฎข้อที่ 31

# Editorial Charter of Islamic Fiqh Publishing (The 51 Iron Rules)

## Section 1: Academic & Editorial Integrity

* **Rule 1:** Maintain 100% of the original content. No truncation or modification of the core substance without editorial approval.
* **Rule 2:** Source code modifications within articles are strictly limited to structural HTML styling and CSS configurations.
* **Rule 3:** No insertion of speculative or unverified data within academic domains (Fiqh, Kalam, Hadith, History).
* **Rule 4:** All analyses must accurately reference classical primary sources (`Primary Sources`).
* **Rule 5:** Avoid colloquialisms or royal terminology when referring to the Prophet and scholars. Use academic prose.
* **Rule 6:** Arabic book titles in references must be preserved in native Arabic script; translation of names is prohibited.
* **Rule 7:** **[Quran Verification]** All Quranic citations must be cross-verified and extracted from `https://quran.com/ar` to ensure flawless diacritics, accompanied by the exact Surah and Ayah number.
* **Rule 8:** **[Hadith Verification]** Hadith texts must be extracted from `https://sunnah.com/` or `https://shamela.ws/` and **must include full Harakat (vowels)**. If unavailable, proceed with the author's original manuscript.

## Section 2: Transliteration & Word Choice

* **Rule 9:** The first mention of a classical scholar must include their demise year: `(d. AH XXX / CE XXX)`.
* **Rule 10:** Islamic honorifics (e.g., Rahimahullah) must follow the name **without parentheses**, even when adjacent to the demise year.
* **Rule 11:** Transliteration of Al- (ال) must conform to phonetic rules, except for "Shari'ah" which must omit the "Ash-" prefix.
* **Rule 12:** Scholarly names must align with standard Arabic phonology (e.g., Al-Ghazali, Al-Maturidi). Avoid the spelling "ท่านอิมาม" or "อิมาม"; enforce "อิหม่าม" (Imam) globally.
* **Rule 12.1:** **[Transliteration of صحيح]** The Thai transliteration of **صحيح** (Sahih) must always be spelled **"ศอเฮี้ยะฮ์"** (e.g., ศอเฮี้ยะฮ์อัลบุคอรีย์, ศอเฮี้ยะฮ์มุสลิม). The spelling **"ศ่อเหี้ยห์" is strictly prohibited.** Apply this consistently across all future content.
* **Rule 13:** **[Transliteration Spacing]** Transliterated religious terminology inserted into Thai sentences must not be preceded or followed by space bars (e.g., `หลักของชะรีอะฮ์ในประเด็น`).
* **Rule 14:** **[Conjunction Choice]** Eliminate archaic conjunctions such as "ทว่า"; enforce the use of "แต่" to optimize academic conciseness.
* **Rule 15:** Direct quotes, Hadith, and translations must be enclosed strictly within curved quotation marks **`"..."`**.
* **Rule 16:** Book titles in texts and bibliographies must be wrapped in `<strong>` tags.
* **Rule 17:** Do not use `<em>` tags or CSS italic styling for book titles.

## Section 3: Language & Typography Structure

* **Rule 18:** The global `<body>` and all Thai elements must state `lang="th"` for accessibility and SEO.
* **Rule 19:** **[Arabic Layout Balance]** Arabic texts extending **≤ 1 line** must be center-aligned. Texts exceeding 1 line must be formatted as standard Blockquotes.
* **Rule 20:** Thai translations of Arabic texts **not exceeding 60 words** must be positioned beneath the source inside `<div class="thai-quote">`. (For texts exceeding 60 words, refer to Rule 49.)
* **Rule 21:** Conceptual keywords highlighted within an essay must strictly use `<strong>` tags.
* **Rule 22:** Fiqh container structures must implement standardized system container classes.

## Section 4: Code & CSS Architecture

* **Rule 23:** CSS rules must be consolidated within `<style>` in `<head>`. Inline `<style>` in `<body>` is prohibited.
* **Rule 24:** **[Font Stack]** Enforce strict typography: **Amiri** for Arabic, **Sarabun** for Thai, and **Inter** for English.
* **Rule 25:** Interfaces must implement `id="header-placeholder"` and `id="footer-placeholder"` for runtime template rendering.
* **Rule 26:** Asset routing in subfolders must apply double-tier backtracking (`../../`).
* **Rule 27:** Implement a typography spacing constraint of `line-height: 2.0` to prevent vowel/diacritic overlap.
* **Rule 28:** **[Thai Typography Fix]** The `text-align: justify;` property is strictly forbidden. Paragraphs must enforce `text-align: left;` paired with `word-break: break-word;` to prevent unnatural white spaces.
* **Rule 29:** Deploy premium gold `#c49a45` for active states and badges to preserve brand identity.
* **Rule 30:** Avoid `Inline Style` unless handling dynamic script-calculated values.
* **Rule 31:** Data arrays for "Quote of the Day" must be structured at the end of the file, preceding `</body>`.

## Section 5: UI/UX & Digital Experience

* **Rule 32:** **[Reading Architecture]** Sidebar TOC systems are abolished. Articles must strictly deploy a **Centered Reading Layout** (max-width: 900px) to maximize reading focus.
* **Rule 33:** Digital utility pages (`tools.html`) must follow a Premium Minimalist aesthetic with subtle box shadows.
* **Rule 34:** Tool modules must render visibility indicators (e.g., "Available", "In Development").
* **Rule 35:** Enable native transition animations on scroll (`Scroll Reveal`) via the `.reveal` class.
* **Rule 36:** Embed a back-to-top component (`#backToTopBtn`) triggered beyond 400px scroll depth.
* **Rule 37:** CSS must support Dark Mode modifications via the `.dark-mode` root class.
* **Rule 38:** The reading tools panel must provide working typography scaling (14px - 28px).
* **Rule 39:** Implement a dynamic Reading Progress Bar pinned at the top viewport boundary.
* **Rule 40:** **[Citation Standard]** Automated bibliographical citations must generate exact format: `Author/Editor. (Year). Article Title. Islamic Fiqh Publishing. Retrieved from [URL]`.
* **Rule 41:** Copy execution buttons must trigger a "Copied ✓" micro-interaction state for 2 seconds.
* **Rule 42:** All static graphics must declare descriptive `alt` attributes.
* **Rule 43:** Formatting grids managing citations in Fiqh articles must separate raw Arabic blocks from localized Thai text, in accordance with Rule 20 (texts ≤ 60 words) and Rule 49 (texts > 60 words).
* **Rule 44:** Title tag nomenclature must maintain the pattern: `[Article Title/Page Title] — Islamic Fiqh Publishing`.
* **Rule 45:** Visual grids must undergo multi-viewport layout validation (`@media`) for mobile compatibility.
* **Rule 46:** The entry script (`index.html`) must allocate a "Featured Applications" grid above articles.

## Section 6: Right-to-Left (RTL) Core Specification

* **Rule 47:** **Arabic Text Direction Standard** To prevent punctuation displacement and ensure accurate bullet point positioning, the `dir="rtl"` attribute must always be enforced alongside `lang="ar"` on all block-level or list elements enclosing pure Arabic text.
* **Rule 48:** **[Blueprint]** Standard Code Blueprint for Block Quotes & Bibliographies (As defined in Thai section).

## Section 7: Long-form & Mixed Script Standards

* **Rule 49:** **[Long-form Source Attribution Standard]** When source text (Arabic or other languages) exceeds **60 words**, apply the following:
  1. Display only the **Thai translation** as the primary content, preceded by a mandatory **attribution formula** by source type, to clearly distinguish the speaker from the article author:
     - **Scholar Quote:** `[Name] (d. AH XXX / CE XXX) [Honorific] said:` e.g. `Al-Subki (d. AH 771 / CE 1370) Rahimahullah said:`
     - **Quranic Verse:** `Allah Subhanahu wa Ta'ala says in Surah [Name], Ayah [X]:`
     - **Hadith:** `The Prophet Muhammad ﷺ said:` (Recorded by [Collector, e.g., Al-Bukhari, Muslim])
  2. Conceal the original source text within a `<details>/<summary>` element labeled **"ดูตัวบทต้นฉบับ ▼"**, positioned beneath the Thai translation within the same blockquote.

* **Rule 50:** **[Inline Arabic Balance Standard]** When Arabic characters appear inline within Thai text, visual balance must be enforced via CSS:
  - **Font size ratio:** Set `font-size: 1.15em` for `:lang(ar)` to visually equalize Amiri with Sarabun.
  - **Vertical alignment:** Set `vertical-align: middle` on `span.ar-inline` to align baselines.
  - **Line height:** Set `line-height: 2.4` on paragraphs containing inline Arabic to prevent diacritic overlap.

* **Rule 51:** **[QUOTE OF THE DAY Standard]** The QOTD system must comply with the following requirements:
  - **Rotation interval:** `setInterval` is fixed at **9,000ms (9 seconds)**. Do not change this value without editorial approval.
  - **Arabic text standard:** Every Arabic entry must carry full Harakat (diacritics), with `lang="ar" dir="rtl"` on the containing element. Quotation marks and periods (`.`) are strictly prohibited inside the `ar` field.
  - **[Critical] Text completeness & attribution:** If the author/editor **explicitly provides the text**, accept it as-is without searching or altering the wording — but still: (1) **add full Harakat** if not present, and (2) **cite the source reference** (book title / database). If no text is provided, conduct a web search against primary sources to verify the text is not truncated, contains no spurious words, and that the attribution is accurate.
  - **Thai translation standard:** Translations must use natural language matching the tone of the source (poetry, literature, theology, or political writing) — accurate, easy to understand, and concise. Use `\n` at natural sentence-break positions to control mobile line wrapping.
  - **Mobile layout:** Use `<br>` instead of `&nbsp;` to break Arabic poetic hemistiches onto separate lines for small screens (except where tested and confirmed to display correctly without breaking).
  - **Data structure:** The `quotes` array must contain `ar` (Arabic text — no quotation marks as content), `th` (Thai translation wrapped in `"..."`), and `author` (`Name · เสียชีวิตปี ฮ.ศ. XXX` — no quotation marks), positioned before `</body>` per Rule 31.
