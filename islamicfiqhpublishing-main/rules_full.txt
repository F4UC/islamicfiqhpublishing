* กฎข้อที่ 34: การจัดกล่องเนื้อหาอ้างอิง (Quote/Hadith) ในบทความนิติศาสตร์ ต้องใช้โครงสร้างควบคุมที่แยกแยะระหว่างตัวบทอาหรับบริสุทธิ์และส่วนแปลไทยเด็ดขาดชัดเจน
* กฎข้อที่ 35: การตั้งชื่อหน้าเว็บในแท็ก <title> ต้องเป็นไปในแนวทางเดียวกัน คือ [ชื่อบทความ/ชื่อหน้า] — Islamic Fiqh Publishing โดยใช้เครื่องหมายขีดข้างยาว (Em Dash)
* กฎข้อที่ 36: โครงสร้าง Layout ต้องได้รับการทดสอบและเขียน Media Queries (@media) ให้แสดงผลแบบ Responsive ได้อย่างสมบูรณ์แบบบนอุปกรณ์สมาร์ตโฟนและแท็บเล็ต
* กฎข้อที่ 37: หน้าแรก (index.html) ต้องออกแบบโครงสร้างสถาปัตยกรรมให้มีส่วนดึง "แอปพลิเคชันแนะนำ" (Featured Apps Grid) แสดงผลในตำแหน่งก่อนเข้าสู่หมวดหมู่บทความล่าสุด
หมวดที่ 6: มาตรฐานการจัดการทิศทางข้อความอาหรับ (RTL Core Specification)
* กฎข้อที่ 38: มาตรฐานการกำหนดทิศทางข้อความอาหรับ (Right-to-Left / RTL)
"เพื่อป้องกันปัญหาการแสดงผลเครื่องหมายวรรคตอนผิดเพี้ยน (เช่น จุดมหัพภาคกระโดดไปอยู่ผิดฝั่ง) และเพื่อให้สัญลักษณ์นำสายตา (Bullet Points) อยู่ในตำแหน่งที่ถูกต้อง ให้บังคับใช้แอตทริบิวต์ dir="rtl" เสมอ ควบคู่กับ lang="ar" ในแท็กระดับบล็อก (Block-level) หรือรายการ (Lists) ที่บรรจุข้อความภาษาอาหรับล้วน เช่น บรรณานุกรมภาษาอาหรับ หรือกล่องอ้างอิง"
ตัวอย่างโค้ดมาตรฐานระดับบล็อกอ้างอิงและบรรณานุกรม:
HTML
<ul class="ref-list" lang="ar" dir="rtl">
   <li>...เนื้อหาภาษาอาหรับ...</li>
</ul>

<blockquote lang="ar" dir="rtl" class="quote-container">
   <span class="ar">...เนื้อหาภาษาอาหรับ...</span>
</blockquote>

<div class="references">
 <h2 class="ref-title" lang="th">บรรณานุกรม</h2>
 <ul class="ref-list" lang="ar" dir="rtl">
   <li>ابن كثير، عماد الدين إسماعيل. <strong>البداية والنهاية</strong>. بيروت: دار إحياء التراث العربي.</li>
   <li>السبكي، تقي الدين. <strong>فتاوى السبكي</strong>. بيروت: دار المعرفة.</li>
   <li>المطيعي، محمد بخيت. <strong>إرشاد أهل الملة إلى إثبات الأهلة</strong>. مصر: المطبعة السلفية.</li>
   <li>الرملي، شمس الدين محمد. <strong>نهاية المحتاج</strong>. بيروت: دار الفكر.</li>
   <li>البخاري، محمد بن إسماعيل. <strong>صحيح البخاري</strong>. بيروت: دار طوق النجاة.</li>
 </ul>
</div>

🌐 Editorial Charter of Islamic Fiqh Publishing (The 38 Iron Rules)
Section 1: Academic & Editorial Integrity
   * Rule 1: Maintain 100% of the original content. No truncation, summarizing, or modification of the core substance is permitted without editorial approval.
   * Rule 2: Source code modifications within article files must strictly be limited to structural HTML styling and CSS layout configurations to ensure accurate rendering.
   * Rule 3: No insertion of speculative, unverified, or unreferenced data within the academic domains of Islamic jurisprudence (Fiqh), theology (Kalam), Hadith, or history.
   * Rule 4: All article content and academic analyses must accurately reference classical primary sources (Primary Sources) according to rigorous research methodologies.
   * Rule 5: Avoid colloquialisms or royal terminology when referring to the Prophet and classical scholars. Utilize precise academic prose.
   * Rule 6: Arabic book titles within the references section must be preserved in their native Arabic script. Transliteration or translation of names is prohibited to maintain bibliographic authenticity.
Section 2: Transliteration & Honorifics Standards
   * Rule 7: The first mention of any classical scholar must include their demise year in parentheses, adhering to the standardized format: (d. AH XXX / CE XXX).
   * Rule 8: Islamic honorifics (e.g., Rahimahullah, Radiyallahu 'anhu) must immediately follow the scholar's name without parentheses, even when positioned adjacent to the demise year.
   * Rule 9: Transliteration of Arabic nouns containing the definite article Al- (ال) must conform to sun-and-moon letter phonetics, except for the word "Shari'ah" which must be written without the "Ash-" prefix.
   * Rule 10: Scholarly technical terms and names of jurists must be transliterated in compliance with standard Arabic phonology (e.g., Al-Ghazali, Al-Maturidi, Al-Subki).
   * Rule 11: Book and treatise titles in both the main text and bibliographies must be wrapped in <strong> tags to emphasize bibliographic metadata.
   * Rule 12: Do not use <em> tags or CSS italic styling for book titles; italics are strictly reserved for general contextual emphasis.
Section 3: Language & Typography Structure
   * Rule 13: The global <body> tag and all Thai textual elements must explicitly state the lang="th" attribute to ensure web accessibility and search engine compliance.
   * Rule 14: Arabic text exceeding a single sentence must not be inserted inline within a Thai paragraph. It must be isolated into a distinct block-level quotation element.
   * Rule 15: The Thai translation of an Arabic quote must be positioned beneath the source text inside a specialized <div class="thai-quote"> element.
   * Rule 16: Important conceptual keywords highlighted within an essay paragraph must strictly implement <strong> tags; no other highlighting mechanics are permitted.
   * Rule 17: Structural frameworks regulating the relationship between Arabic text inputs and Thai equivalents must implement standardized system container classes.
Section 4: Code & CSS Standards
   * Rule 18: Universal CSS rules must be consolidated within internal <style> blocks in the <head> infrastructure. Inline styling sections inside the <body> are strictly prohibited.
   * Rule 19: All document interfaces must implement id="header-placeholder" and id="footer-placeholder" nodes to allow the asynchronous runtime template rendering managed by main.js.
   * Rule 20: URL target references and static asset routing definitions within subfolders must uniformly apply the relative double-tier directory backtracking pattern (../../).
   * Rule 21: Implement a default typography spacing constraint of line-height: 2.0 across all long-form articles to prevent overlap between Thai vowels and Arabic diacritics.
   * Rule 22: Deploy the signature premium gold color token #c49a45 for application badges, active interactive state components, and buttons to preserve visual brand identity.
   * Rule 23: Avoid using Inline Style attributes on HTML elements unless handling dynamic values calculated directly by script routines.
   * Rule 24: Data arrays and logical scripts for the "Quote of the Day" mechanism must be cleanly structured at the end of the markup file, preceding the closing </body> tag.
Section 5: UI/UX & Digital Experience
   * Rule 25: Digital utility pages (tools.html) must follow a Premium Minimalist design aesthetic, featuring clean white bordered card modules with subtle box shadows.
   * Rule 26: Every tool and application module card must render a clear visibility indicator via text labels (e.g., "Available", "In Development", or "Under Evaluation").
   * Rule 27: Enable native element transition animation systems on window view movement events (Scroll Reveal) by appending the .reveal animation handler class.
   * Rule 28: Embed a smooth back-to-top navigational anchor component (#backToTopBtn) configured to toggle on when document scrolling offsets exceed 400 pixels.
   * Rule 29: CSS structural layouts must inherently support system theme state modifications (Dark Mode / Night Mode) controlled via the global toggle class .dark-mode on the root node.
   * Rule 30: The reading tools panel must provide a working typography scaling feature (A-, Normal, A+) restricted within the bounds of 14px and 28px.
   * Rule 31: Implement a dynamic linear page scroll tracking bar (Reading Progress Bar) pinned at the top viewport boundary of all text essays.

Section 6: Right-to-Left (RTL) Core Specification
   * Rule 38: Arabic Text Direction Standard (Right-to-Left / RTL)
"To prevent punctuation displacement issues (such as periods leaping to the wrong side of the sentence) and to ensure that list indicators (Bullet Points) are correctly positioned, the dir="rtl" attribute must always be enforced alongside lang="ar" on all block-level or list elements enclosing pure Arabic text blocks, including Arabic bibliographies and reference components."
